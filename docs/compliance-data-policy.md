# CampusFit AI 合规与数据处理基线

## 数据分类

- 账号数据：邮箱、登录会话、账号状态。
- 身体数据：身高、体重、出生年、训练经验、目标类型、活动水平。
- 饮食数据：饮食场景、饮食偏好、饮食限制、三餐记录。
- 训练数据：训练计划、动作、组数、休息、打卡完成度。
- AI 数据：对话内容、引用、trace、上下文。

## 最小化原则

- 页面只展示完成用户任务所需数据。
- 日志只记录请求路径、状态、耗时和 request id，不记录 Cookie、token、密码、OpenAI key、身体数据明细。
- 健康检查不得暴露内部服务地址、密钥或连接串。

## 数据删除流程

1. 用户在个人中心点击“申请删除数据”。
2. API 写入 `data_deletion_requests`，状态为 `pending`。
3. 运营人员核验申请人身份和账号风险。
4. 删除或匿名化用户相关的训练、饮食、打卡、AI 对话、身体数据。
5. 对依法或安全审计必须保留的记录，只保留最小必要字段并脱敏。
6. 完成后把申请状态更新为 `processed`，并记录处理时间。

## 用户告知

- 公开页面必须提供隐私政策、用户协议和数据删除说明。
- 用户协议必须明确：训练和饮食建议不替代医疗建议。
- 隐私政策必须明确：身体数据用于计划生成、执行提醒和产品体验，不用于无关用途。

## 上线前责任

- 产品负责人复核页面文案。
- 运维负责人确认备份、恢复、告警和日志脱敏。
- 技术负责人确认鉴权、CORS、Swagger、生产密钥校验、CI 质量门禁。
