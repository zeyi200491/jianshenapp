# CampusFit AI 页面树

## 1. 页面设计原则

1. 公开网页是 MVP 主入口，页面结构优先服务于每日执行闭环。
2. 页面层级控制在浅层，避免多次跳转。
3. 所有页面必须具备 `loading / empty / error` 三种基础状态。
4. 核心动作优先暴露在首屏和侧边导航 / 顶部导航附近。
5. 页面必须兼容桌面与移动浏览器。

## 3. 导航建议

MVP 建议采用左侧导航或顶部主导航，包含：

1. 今日
2. AI 助手
3. 周复盘
4. 商品
5. 个人中心

说明：

- 打卡作为今日页核心动作，不单独放主导航。
- 饮食与训练详情通过今日页进入。
- 首页承担产品说明与登录转化。

## 4. 页面职责说明

### 4.1 首页 / 落地页

- 说明产品价值
- 承接 SEO / 分享流量
- 引导登录与开始建档

### 4.2 登录页

- 邮箱验证码登录
- 检查是否已建档
- 未建档跳转建档向导

### 4.3 建档向导

- 分步收集用户信息
- 每步支持返回修改
- 完成后触发初始计划生成

### 4.4 今日页

- 作为最高频访问页面
- 展示今日计划摘要与完成状态
- 提供打卡、问答、商品入口

### 4.5 饮食计划详情页

- 展示餐次建议和替代方案
- 解释每餐目标
- 支持饮食完成打卡

### 4.6 训练计划详情页

- 展示动作与组次
- 展示注意事项
- 支持训练完成打卡

### 4.7 打卡页

- 收集当日记录行为
- 一次性提交饮食、训练、体重和主观状态

### 4.8 周复盘页

- 每周固定生成一份复盘
- 以趋势、亮点、问题和建议四段结构输出

### 4.9 AI 助手页

- 默认注入用户档案和当日计划摘要
- 用于回答执行相关问题
- 非关键路径，出错时可降级为静态引导

### 4.10 商品列表页

- 仅展示基础商品信息
- 先支持分类、标签、适用场景
- 交易闭环延后

## 5. 运营后台页面建议

├─ 训练模板管理
